Database Driven Pipelines

Databases gives speed and flexibility for the whole team: Fast artist tools, Complicated present and future predictions can be made with historical data available. Good databases I like to use are usually MongoDB and MySQL databases development, replication, backups, optimization. MySQL is create for relational data where MongoDB for fast and unsorted/dynamic data.

File System IT & Cleanup

Very few people like to clean and even fewer like to clean other people belongings. Plus it takes time and a good understanding of the data you're about to clean. For that reason, it's best to structure the files/data in a way that makes it easier to clean later. Or even better, deploy smart and automated cleaning process. All of which to help backup, archive and monitoring of data.

VM's & Dockers Services

Pipeline and IT using many services in the backend to help productions. Deploying and managing these services can be by deploying services using ESXi, Virtual machines, Docker.

Render Farm Deployment and Management

Management, deploying and monitoring of render-farm can be done smartly, using automated emails/alert, production-driven queuing systems, expanding to the cloud and overall making wrangler life easier.

Shotgun Integration & Advance Dashboards

Integrating Shotgun into the artist's workflow helps the production team to be on top of everything. Integration in tools, workflows, triggers for automation. While Shotgun pages are great, many studios end-up building custom Dashboard to better monitor their productions, with more complicated queries and tracking..

Resources Monitoring & Metrics

Monitoring and building metrics of the studio resources is key for savings and smart growth. Knowing the numbers behind licenses, servers, workstations helps so much. Using Grafana, Prometheus and Sentry services to collect and monitor backend/frontend services.

Rendering Optimizations

There are many great render engines out there. Each has requires optimizations and a pipeline around it to push the render times down. Optimizations are usually caching assets, creating proxies, pre-converting textures. .

Asset Management: Proprietary NextGen Framework

Developing proprietary asset-frameworks is one of my passions. I put a lot of love and a rich feature set in those. To name a few of the features: Agnostic to DCC/OS. Each publish is a version, each version can be linked to other versions/asset/take, each version has a metadata, supports approval/rejection. supports smart disk-space cleanup, post-processing, farm publishing, pre-publish asset checks... and much more.

Color Accurate From Start to End

It can be somewhat "boring" area in production... having a colour-accurate pipeline from end to end is the only way to guarantee quality delivery of images. Images coming out of Photoshop should flow to rendering with the correct colorspace, with as little assumptions made on the way. Add to that calibrated monitors and you're in business!.

Custom Artists Tools For All DCCs

Artists need custom tools and support. Many steps and processes can be simplified. Free the artists do focus on "art" and less fighting tech challenges along the way, that makes a good goal. All of the DCC could use more tooling: Maya, Nuke, Katana, Unity, Mari, Zbrush, Mudbox...
